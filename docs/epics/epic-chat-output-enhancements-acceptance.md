### 4.x 已知问题

**代码语法高亮在真实 LLM 输出中不生效**：GLM-5.1 模型流式输出时将 ` ```swift ` 拆为两行（` ``` ` + `swift`），导致 `StreamingCodeBlockRenderer` 把 `swift` 识别为代码内容而非语言标签。单元测试和 E2E 测试中手动构造的输入是单行 ` ```swift ` 所以能通过。

**修复建议**：在 `processCompleteLine` 的 `inCodeBlock` 分支中，若 `currentLang` 为空且是第一行内容，检测该行是否仅含语言标签字符串，如果是则设为 `currentLang`。
