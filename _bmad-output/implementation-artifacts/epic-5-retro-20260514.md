### 突破时刻

**1. Hummingbird ResponseBody AsyncSequence for SSE**
发现 Hummingbird 2.x 原生支持 `ResponseBody(asyncSequence:)` 构建 SSE 流，无需外部库：
- 大幅简化了 SSE 实现
- 性能更好（无中间层）

**2. tryAcquire + Background acquire 模式**
Story 5.3 的 review 发现 acquire() 会阻塞 HTTP handler，创造性地引入：
- `tryAcquire()` — 非阻塞检查，立即返回
- 后台 `Task { await acquire() }` — 排队等待
- **启示：** 当异步原语与 HTTP handler 交互时，非阻塞检查 + 后台等待是更好的模式
